1. Setting up

1.1 - Installing R

Before starting anything, you’ll need to download two applications:

  • Download R from the link here
  • Download the integrated developer environment (IDE) RStudio here

As all data analyses conducted were written in R. When done, open up RStudio.

Once RStudio has been opened, please familiarise yourself with the IDE and syntax. R is an open source programme and is highly flexible, so I would definitely recommend learning a little more about this language! Helpful tutorials can be found here or here.

1.2 - Installing Libraries

Many R codes begin with importing libraries, which look like this:

library(ggplot2)
library(readr)
library(dplyr)

You’ll need to download the libraries into your computer on your very first time using RStudio, as it’ll subsequently already be installed the next time you use R. To download the library, type install.packages("ggplot2") into the console window (below), for example to download package ggplot2.

1.3 - Creating a default graph folder

All of the R codes written here will look for a default folder to automatically export all the graphs to. To help facilitate the proper function of the code, please do the following before attempting to run any code:

On the desktop, create a new folder called “Graph Directory” (must be capitalised!)

Any time you re-run the code, R will automatically save over the old graphs only if they share the same title, hence be careful when running the same code twice! Once you have run the code and produced the graphs, extract the graphs from the directory by copy-pasting all the relevant graphs to another folder so that you don’t lose the data.

2. Running R

2.1 - Merging the source file

The codes are designed to read one merged file (be it a .csv or .log file). First and foremost, copy all the files of the same type (i.e. portable data should not be mixed with bus stop data) into the same folder. Let’s presume that we’ve copied all the files into a folder called [combine] in the desktop.

The folder’s directory should then be [C:/Users/Public/Desktop/combine]. Next:

  1. Open the command prompt (type cmd into the desktop search bar)
  2. In command prompt, set the command directory to the folder location (type cd C:/Users/Public/Desktop/combine)
  3. In command prompt, combine all the files into a merged file (type combine *.csv mergedfile.csv)
  4. This creates a combined file called [mergedfile.csv]
  5. If the file is a .log file, just replace all the .csv above in steps 1-3 with .log

Now we have one merged file. Next, we need to help R locate this file.

2.2 - Finding the import file location

All the codes have a beginning section that looks like this:

# ================================================

# Please edit both the location of the CSV file (default location being the desktop)
# Also edit the month of the data

fulldata = read_csv("C:/Users/Public/Desktop/combine/mergedfile.csv")
mm = "September" #only change label


# ================================================

This function, read_csv, or read_table for the bus stop PM/Temp file, automatically reads a .csv/.log file extension into a format that R can process. Remember the merged file earlier? To help R find this file, we need to type in the exact file name into the function.

Locate the correct file location and type that into the brackets. To do so quickly, simply right click the name of the folder like:

And select the [Copy address as text] option, then paste it into the file directory in R. Please take note of two common mistakes when copying the file directory:

  1. The file location must be typed between inverted commas (i.e. “C:/Users/” etc. and NOT plainly C:/Users/)
  2. R file locations must be typed in the front slash (i.e. C:/Users/ etc. and NOT C:\Users\etc.)

Don’t forget to also change the label of the month you’re evaluating for the bus_stop files (i.e. change label from September to October as prompted in the code).

2.3 - Running the Code

To run R code, simply click the source button located on the top right section of the RStudio interface and allow the code to run. Wait about 2 minutes (for a data file of 10 million data points) and retrieve the graphs from the graph folder specified above.

If there is any error, RStudio will prompt a warning in the Console interface. Do troubleshoot accordingly, as the errors likely come from any of the mistakes stated earlier.

R is ready to run when you see the > symbol in the Console interface. When R is running a code, the console interface will not have the > symbol. To interrupt a code if necessary, click on the ESC key on the keyboard. Once R is done running the code, a prompt saying "Code Complete" should be seen in the command console.
